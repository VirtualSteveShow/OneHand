| ✅ Done | Fifth game — Sling | Built 2026-07-05. New input primitive at `public/games/sling/`: pull-back-and-release. Press anywhere to plant an anchor at the touch point (floats like Orbit's wheel), drag away from it to pull back a rubber-band, release to launch a ball on a gravity arc — direction AND pull distance both matter, the first mechanic here to encode two continuous values in one drag instead of one. A dotted trajectory preview renders while aiming. Aiming is by pulling away from the target, never by touching it, so all input stays low on screen regardless of where the target sits (a design constraint the user flagged: the thumb can only reach the bottom of the screen while holding a baby in the other arm). Best score in localStorage. Verified with Playwright using real touch press/drag/release — confirmed real gravity-driven flight, a genuine hit via a target placed at the ball's actual mid-flight position, and a genuine miss by pulling the opposite direction to send the ball off the bottom of the screen. **Fixed 2026-07-05:** real-device testing showed some targets were unreachable even at max pull (reachable envelope math: max height was only `v²/2g` ≈ 278px with the original 850 px/s speed / 1300 px/s² gravity, far short of the ~700px+ climb needed from a bottom-anchored shot to a top-corner target). Reworked to 1400 px/s max speed / 1000 px/s² gravity, verified against the worst-case corner-to-corner case with the actual "parabola of safety" formula (`Y_up <= v²/2g - g·dx²/2v²`) plus a live Playwright test computing the exact launch angle and confirming a real hit. |
| ✅ Done | Sixth game — Tilt (gyro) | Built 2026-07-05. New input primitive at `public/games/tilt/`: hold + tilt. Press and hold anywhere (no drag, just touch presence) arms the device's gyro; while held, tilting the phone left/right steers a ball continuously (not discrete lanes) to dodge single falling blocks (same one-at-a-time fairness model as Dash/Orbit). Releasing freezes the ball in place. Addresses the CLAUDE.md caution about tilt input (a baby-holding hand rocks unpredictably) with two mitigations: the gyro's `deviceorientation` value is only ever read while a finger is actively down (`gyroActive` gate), and every single press recalibrates a fresh zero-point from whatever angle the phone is at that moment (`zeroGamma`), so an unusual resting grip or drift between plays never becomes false input — only deliberate tilting relative to your own current grip moves the ball. Also handles iOS's `DeviceOrientationEvent.requestPermission()` prompt (required before iOS 13+ Safari will deliver motion events at all; Android needs no such prompt), with a "MOTION ACCESS NEEDED" message shown on the start screen if permission is denied. Best score in localStorage. Verified with Playwright dispatching synthetic `deviceorientation` events — confirmed a pre-press baseline tilt is correctly ignored (calibrated away), symmetric left/right steering response, freeze-on-release, and a genuine forced collision. **Known test gap:** Playwright/Chromium has no `requestPermission` (that's WebKit/iOS-only), so the iOS permission-prompt path itself couldn't be exercised in automated testing — needs a real iOS device check. |
